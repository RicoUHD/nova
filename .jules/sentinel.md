## 2025-02-14 - Fix Stored XSS via File Upload
**Vulnerability:** The `/api/upload` endpoint used a `multer` `fileFilter` that only checked the `file.mimetype` against a whitelist of allowed image types. It did not validate the actual file extension (`file.originalname`).
**Learning:** A malicious user could upload a file with an `.html` extension (e.g., `exploit.html`) but spoof the `Content-Type` header to `image/jpeg`. `multer` would accept it based on the spoofed MIME type and save it with the `.html` extension. When another user downloads the file via `/api/receipts/:filename`, the server uses `res.sendFile()`, which infers the `Content-Type: text/html` from the `.html` extension, causing the browser to render the file as an HTML page and execute any embedded malicious scripts (Stored XSS).
**Prevention:** Always validate both the MIME type and the file extension against a strict allowlist. Ensure that files intended to be images are only saved and served with safe image extensions (`.jpg`, `.png`, etc.) to prevent browsers from executing them as code.
